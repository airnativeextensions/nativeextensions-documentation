Property Detail
isSupportedproperty
isSupported:Boolean  [read-only]

Whether the current device supports BluetoothLE advertising, i.e. acting as a peripheral

This flag is supplied to correctly handle devices that don't support Bluetooth LE peripheral mode such as Android version pre 5.0

If this returns false then none of the functionality provided by the PeripheralManager is supported

Method Detail
addService()method
public function addService(service:Service):Boolean

Registers the service with the devices database of published services.

Note that you cannot change the service or any of it's characteristics after you publish the service. You should still hold onto the reference of the service and characteristic to respond to read and write requests.

You should listen for PeripheralManagerEvent.SERVICE_ADD and PeripheralManagerEvent.SERVICE_ADD_ERROR to determine if the publishing of this service was successful.

Parameters

service:Service — The service including all characteristics to publish.

Returns
Boolean — true if the service publish was started successfully and false if not, generally if the bluetooth hardware is in an invalid state.

respondToRequest()method 
public function respondToRequest(request:Request, result:String, value:ByteArray = null):Boolean

You should call this function exactly once for each RequestEvent.READ or RequestEvent.WRITE received.

This function responds to a read or write request with the specified result and value.

Parameters

request:Request — The request object to respond to (RequestEvent.request)
 
result:String — The result code to send. Should be one of the defined values in the Request class, eg. Request.SUCCESS
 
value:ByteArray (default = null) — The ByteArray data to respond to read requests with. You should make sure you have obeyed the offset provided in the request.

Returns
Boolean — true if successful and false if the adapter was in an invalid state
