enableNotificationsWhenActiveproperty 
public var enableNotificationsWhenActive:Boolean = false

By default this is false and a notification delivered when the application is open will not be shown to the user, and instead the notification event is fired. If set to true notifications will be shown when the user is in your app.

Supported on:

  • Android
  • iOS v10+

locationSharedproperty 
public var locationShared:Boolean = true

For services that support location based targetting this flag disables or enables the collection of the user location.

Defaults to enabled however is also dependant on your application having permission to access the device location.

This is supported on the following services:

  • OneSignal

requiresUserPrivacyConsentproperty 
public var requiresUserPrivacyConsent:Boolean = false

Lets your app require the user's privacy consent before it will initialize. If you set this to true, the SDK will delay initialization until your application calls provideUserConsent(true).

If the SDK is waiting for the user's consent, calling any methods will do nothing but print a warning. The user will not be registered for push notifications until this happens.

This is supported on the following services:

  • OneSignal

See also

com.distriqt.extension.pushnotifications.PushNotifications.provideUserConsent()

serviceIdproperty 
public var serviceId:String

A service specific identifier or reference.

The service Id is used in the registration process to identify a 3rd-party application server that is permitted to send messages to the device.

With GCM this will the sender ID associated with the applications API Project. This is a project number you acquire from the API console, as described in the GCM API Project "Getting Started" section.

With Azure this will be the "hubname"

serviceKeyproperty
