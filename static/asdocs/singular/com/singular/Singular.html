revenue()method 
public function revenue(currency:String, amount:Number, productSKU:String = null, productName:String = null, productCategory:String = null, productQuantity:int = -1, productPrice:Number = -1):Boolean

Send a revenue event to Singular with optional additional information Singular can collect data about revenue gained through the app to help analyze the performance and ROI of your campaigns. Singular will make the data available to you in reports, log export, and postbacks. Note that when reporting revenue events this way, Singular does not get the purchase receipt and cannot validate the transaction.

Parameters

currency:String — Pass currency as a three-letter ISO 4217 currency code, such as “USD”, “EUR”, or “INR".
 
amount:Number — Amount of the purchase
 
productSKU:String (default = null) — Optional: product identifier / sku
 
productName:String (default = null) — Optional: product name
 
productCategory:String (default = null) — Optional: product category
 
productQuantity:int (default = -1) — Optional: product quantity
 
productPrice:Number (default = -1) — Optional: product price

Returns
Boolean — true on success and false otherwise

public function stopAllTracking():void

Stop all tracking activities for this user on this app. Note: Calling this method effectively disables the SDK, even after the app restarts (the state is persistent)! The only way to re-enable tracking is by calling resumeAllTracking().
