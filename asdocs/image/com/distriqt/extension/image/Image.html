Packagecom.distriqt.extension.image
Classpublic class Image
InheritanceImage flash.events.EventDispatcher

This class represents the image extension.

Encoding

A major feature of this extension is the native encoding of bitmap data allowing you to quickly encode a bitmap to a PNG or JPG.

The most simple usage is to call the encode function that takes a BitmapData object and encodes it to a ByteArray containing the encoded bytes in the specified format. 

     Image.service.encode( bitmapData, encodedData, ImageFormat.JPG, 0.5 );

public function saveBitmapDataAsync(bitmapData:BitmapData, filename:String, format:String, quality:Number = 0.8, cacheBitmapInternally:Boolean = true):Boolean

Save bitmap data to a file on the device asynchronously.

This will use the devices native functionality to encode and save the file. The native encoding is a lot faster than using the equivalent AS3 encoder.

If you are concerned about the speed of this function you should try to set cacheBitmapInternally to false and make sure that you hold onto a reference to your bitmap data.

Parameters

bitmapData:BitmapData — The bitmap data to save to a file
 
filename:String — The full path to the file to save
 
format:String (default = NaN) — The format of the file, PNG, JPG etc
 
quality:Number (default = 0.8) — Quality of the encoding (must between 0 - 1)
 
cacheBitmapInternally:Boolean (default = true) — If true (default) the extension will make a copy of the bitmap data internally so that if AIR disposes the data the extension won't crash. If you are holding on to a reference to the Bitmap you can set this to false to conserve memory usage.

Returns
Boolean — true if operation was started successfully, and false if there was an error starting or if there is another operation already in progress

transformAndSaveAsync()method 
public function transformAndSaveAsync(image:BitmapData, transform:ImageTransform, output:ImageOutput, callback:Function = null, cacheBitmapInternally:Boolean = true):Boolean

You can use this function to transform a BitmapData instance, by applying a resize algorithm. The result is saved to the specified output file.

Parameters

image:BitmapData — BitmapData of the image to transform
 
transform:ImageTransform — The transformation to use on the image
 
output:ImageOutput — The information on the output file (including filename and format)
 
callback:Function (default = null) — A function to call when the transform and save completes
 
cacheBitmapInternally:Boolean (default = true)

Returns
Boolean — true if the transform process started successfully and false if there was an issue starting the transform

Example
The following resizes the image to be best fit within 300x300 pixels This will keep the aspect ratio the same and scale the image so that the larger dimension is 300 pixels. 
         var image:Bitmap = ...;
                  var transform:ImageTransform = new ImageTransform();
         transform.resizeMethod = ImageTransform.RESIZE_BEST_FIT;
         transform.width = 300;
         transform.height = 300;
                  var file:File = File.applicationStorageDirectory.resolvePath("images/output.jpg");
                  var output:ImageOutput = new ImageOutput( file.nativePath );
         output.format = ImageFormat.PNG;
                  function savedCallback( success:Boolean ):void
         {
             // transformAndSaveAsync has completed
         }
                  if (!Image.service.transformAndSaveAsync( image.bitmapData, transform, output, savedCallback ))
         {
              trace( "Error starting transform" );
         }

Examples

The following example encodes bitmap data into the specified format and returns the data stream. It then demonstrates using the data by loading it onto the screen again, though in reality you're more likely to send it to a server or save it to a file. 

     // Create some test bitmap data
     [Embed("test.png")]
     public var Icon:Class;
          ...
          var icon:Bitmap = new Icon() as Bitmap;
               // Destination byte array
     var encodedData:ByteArray = new ByteArray();
               Image.init( DEV_KEY );
     Image.service.encode( icon.bitmapData, encodedData, ImageFormat.JPG, 0.5 );
               //
     // Demonstrate usage by loading the bytes
          var loader:Loader = new Loader();
     addChild( loader );
          loader.loadBytes( encodedData );
