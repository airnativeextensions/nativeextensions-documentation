setDoNotSell()method 
public function setDoNotSell(isApplied:Boolean, consentString:String):Boolean

CCPA - Privacy String The intention of the California Consumer Privacy Act of 2018 (CCPA) is to protect the personal information of California residents. CCPA applies to all companies doing business in California. If a California resident uses an app developer’s mobile app, CCPA applies to the developer and every company that processes the personal information of the app’s users. The CCPA requires you to collect your user's consent to collect their data for users within the region scope. Once you have collected the user’s consent, you can pass it onto the SDK using the following API

Parameters

isApplied:Boolean — Whether the user has accepted the CCPA
 
consentString:String — The consent string

Returns
Boolean — true if successful, false otherwise
setIsChildDirected()method 
public function setIsChildDirected(isChildDirected:Boolean):Boolean

COPPA Compliance (Child-Directed Apps with Age Gates) Odeeo’s SDK (starting SDK Version 1.4.0+) enables publishers to flag specific end-users as children, as may be permitted or required by applicable law. Publishers of child-directed apps are responsible for determining whether an app is permitted to flag at the end-user level or must treat all end-users as children. Publishers should consult with their legal counsel accordingly. If your app is entirely directed to children or entirely not directed to children, this can be placed as a global flag on Odeeo’s end (please approach your account manager for this setup). Otherwise, you may choose to send this indication on a user-level basis via the SDK function. The indication of whether a specific end-user is a child should be done using a “IsChildDirected” flag, by setting its rewardValue to “true” or “false” as detailed below

Parameters

isChildDirected:Boolean — Whether the user is a child or not

Returns
Boolean — true if successful, false otherwise

setPublisherUserID()method 
public function setPublisherUserID(userId:String):Boolean

Set an internal user ID for reporting an attribution reasons. This ID may be your generated ID, or your MMP user ID - depending on your reporting needs. This ID can later on be used for reporting revenue per user to your desired platform (for example - ILRD reporting to your MMP for LTV calculation). It is recommended to pass this ID right after initialising the SDK in order to have the relevant ID for the entire user session.

Parameters

userId:String — The user ID to set

Returns
Boolean — true if successful, false otherwise
