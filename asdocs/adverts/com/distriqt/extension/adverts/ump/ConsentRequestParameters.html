setConsentSyncId()method 
public function setConsentSyncId(consentSyncId:String):ConsentRequestParameters

An optional string that can be used to identify a user across devices and platforms. This is used for syncing consent status across devices and platforms. The identifier you provide must uniquely identify the user across all of your apps where consent is being synced. Hash or encrypt the identifier to prevent sending personally identifiable information (PII) to Google. The provided ID must meet the following requirements:

  • Constructed as a UUID string or matches regular expression ^[0-9a-zA-Z+.=\/_\-$,{}]{22,150}$.
  • A minimum of 22 characters.
  • A maximum of 150 characters.
The following are examples of correct consent sync IDs:
  • 12JD92JD8078S8J29SDOAKC0EF230337
  • 12jd92jd8078s8j29sdoakc0ef230337
  • 12Jd92jD8078s8j29sDoakc0ef230337
  • 123e4567-e89b-12d3-a456-426614174000
Failure to meet the requirements results in the consent sync ID not being set and the UMP SDK logging a warning to the console.
