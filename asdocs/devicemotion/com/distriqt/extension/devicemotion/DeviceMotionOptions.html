algorithmproperty
public var algorithm:String = native

The algorithm represents the method we use to calculate the position and motion of the device.

This may use a combination of sensors in a 'fusion' or just raw sensor data. See the individual ALGORITHM_ definitions for more details on each algorithm.

See also

Algorithm.ALGORITHM_NATIVE
Algorithm.ALGORITHM_FUSION
autoOrientproperty 
public var autoOrient:Boolean = true

This allows you to control whether the extension will correct for screen rotations.

If true the algorithm will determine the current screen orientation and correct the coordinate system so that the current screen up vector is the y coordinate.

If false the results will always be relative to the default orientation of the screen and you will need to handle this when displaying information.

formatproperty 
public var format:String = quaternion

This controls what data the extension returns when device motion is detected and the type of update events that are fired from the extension. This allows us to optimise the code to only calculate the information required.

You can select euler angles (though we highly suggest you do not use these) or something more reliable such as quaternions

See also

OutputFormat.EULER
OutputFormat.QUATERNION
OutputFormat.ROTATIONMATRIX
rateproperty 
public var rate:int = 3

The rate at which updates should be requested from the sensors. Note that this rate is the query rate of each individual sensor.

See the SensorRate.SENSOR_DELAY_XXX values. Default is SensorRate.SENSOR_DELAY_NORMAL

Sets the reference frame to use for the algorithm. This will influence the sensors that are required to calculate the outputs for the selected algorithm.
