startMonitoringRegion()method 
public function startMonitoringRegion(region:Region, options:LocationRequest = null):Boolean

Starts monitoring a region for entry and exit events.

Once a successful call to this function is made the extension will dispatch a RegionEvent for any changes in the region.

You must correctly set the properties of a region to be able to start monitoring the region. This includes the latitude, longitude, radius and identifier values. You can call the isValid function on the region to check your region is valid.

Note: If you are adding multiple regions you should wait until the RegionEvent.START_MONITORING event is fired before adding another. Adding multiple regions without waiting may cause some of the region monitoring not to correctly start.

Parameters

region:Region — Details on the region to monitor. It's important that you set a unique identifier value on this object.
 
options:LocationRequest (default = null) — Location options, specifying the location options such as accuracy of the monitoring

Returns
Boolean — true if the region was successfully added to the monitored regions
