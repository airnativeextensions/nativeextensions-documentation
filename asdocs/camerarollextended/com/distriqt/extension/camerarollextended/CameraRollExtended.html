Classpublic final class CameraRollExtended
InheritanceCameraRollExtended flash.events.EventDispatcher

This class represents the CameraRollExtended extension.

This extension allows you to present a dialog to the user to select multiple assets from the user's camera roll. Currently the extension supports selecting and loading images but only selecting videos. However you may use the url from the selected video as an asset url in a native video player.

The selection overlays can be customised. See the browseForAsset documentation for more on this.

browseForAsset()method 
public function browseForAsset(options:CameraRollExtendedBrowseOptions = null):Boolean

The main call to display a UI for the user to select multiple assets from their library.

This call will display a native-stlyed user interface dialog to select multiple images and assets. Depending on the method and the device some indicator similar to a small "tick" overlay is shown over the image thumbnail to indicate the image has been selected to the user.

The overlay can be customised only on iOS versions less than 8 by packaging two images with your application with the following filenames and pixel dimensions.

  • Overlay.png (75x75)
  • Overlay@2x.png (150x150)

There are no customisation options for the other platforms apart from what is available in the browse options.

The sample ones packaged with the example application can be used as a template. These should be placed in the root src directory of your AIR application and included in the application package contents.

getNativePathForAsset()method 
public function getNativePathForAsset(asset:Asset):String

Returns the native path for an asset. In some situations, eg playing a video file using the MediaPlayer ANE this path is all you need to present the asset.

The native path may not be accessible via a File reference. This is due to file access permissions on the operating systems.

You can attempt File access and check the exists flag to see if the file is accessible.

Parameters

asset:Asset — The Asset reference to load

Returns
String — The native path string of the specified asset

Example
The following example uses the MediaPlayer ANE to play a video asset 
         var nativePath:String = CameraRollExtended.service.getNativePathForAsset( asset );
         if (nativePath != null && MediaPlayer.isSupported && asset.type == Asset.VIDEO)
         {
             MediaPlayer.service.createPlayer( nativePath, 0, 0, 640, 480, true );
         }

If you load a group array of assets then you'll receive this event once all the assets have been retrieved and loaded from the asset library.

Additionally if you've selected autoLoadBitmapData in the browse options then you'll receive this event once all the selected images have been loaded.

The assets property on this class will contain all the information, including valid bitmap data for all the loaded assets.

This event is dispatched after a user selects one or more images from the gallery.

At this point, the assets property will contain information about the default representation of the selected images, however the bitmap data will be invalid.

If you've selected autoLoadBitmapData then the load process will start after this event.

Examples

This simple example initialises the extension then displays the browse dialog for the user to select multiple images. It also listens for the selection event to process the selected images. 

          CameraRollExtended.init( DEV_KEY );
     CameraRollExtended.service.addEventListener( CameraRollExtendedEvent.CANCEL, cameraRoll_cancelHandler, false, 0, true );
     CameraRollExtended.service.addEventListener( CameraRollExtendedEvent.SELECT, cameraRoll_selectHandler, false, 0, true );
          CameraRollExtended.service.browseForImage();
               ...
               private function cameraRoll_cancelHandler( event:CameraRollExtendedEvent ):void
     {
         // The user cancelled the browse operation
     }
          private function cameraRoll_selectHandler( event:CameraRollExtendedEvent ):void
     {
         for each (var asset:Asset in event.assets)
         {
             trace( "selected: "+ asset.filename + " [" + asset.type + "] -> " + asset.url );
         }
     }

This example presents a browse dialog to the user then auto loads the thumbnails for the selected images and adds the bitmap to the stage when loaded. The dialog is limited to a maximum of two selected images. 

     CameraRollExtended.init( DEV_KEY );
     CameraRollExtended.service.addEventListener( CameraRollExtendedEvent.LOADED, cameraRoll_loadedHandler, false, 0, true );
          var options:CameraRollExtendedBrowseOptions = new CameraRollExtendedBrowseOptions();
     options.maximumCount = 2;
     options.type = Asset.IMAGE;
     options.autoLoadBitmapData = true;
     options.autoLoadType = AssetRepresentation.THUMBNAIL;
          CameraRollExtended.service.browseForImage( options );
          ...
               private function cameraRoll_loadedHandler( event:CameraRollExtendedEvent ):void
     {
         // The selected images have been loaded
         for each (var asset:Asset in event.assets)
         {
             if (asset.bitmapData != null) // Sanity check
             {
                 addChild( new Bitmap( asset.bitmapData ));
             }
         }
     }
