finishMatch()method 
public function finishMatch(results:Array):void

Mark a match as finished.

This should be called when the match is over and all participants have results to be reported (if appropriate).

Note that the last client to update a match is responsible for calling finish on that match.

On the last turn of the match, the client should call this method instead of takeTurn.

In order to finish the match with results you should create an array of ParticipantResult objects for the participants in the match that require a result. You don't have to assign results to participants, however doing so will enable the service to display results to the player. Providing results for participants who are not in the match is an error.

results:Array

takeTurn()method 
public function takeTurn(nextParticipant:Participant):void

Update a match with new turn data. The participant that is passed in as the pending participant will be notified that it is their turn to take action. If no pending participant is provided and the match has available auto-match slots, the match will wait for additional players to be found. If there are no auto-match slots available for this match, a pending participant ID is required.

Note that players will not receive invitations for matches until this method is called. An invitation will be sent to a player the first time they are set as the pending participant of a match.

This will dispatch a TurnBasedMatchEvent with the result, either TAKETURN_SUCCESS and this match will be updated or TAKETURN_FAILED on an error.

Parameters

nextParticipant:Participant — The next player/participant to have their turn
