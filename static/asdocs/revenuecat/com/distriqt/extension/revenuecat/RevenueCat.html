restorePurchases()method 
public function restorePurchases(success:Function = null, failure:Function = null):void

Restoring purchases is a mechanism by which your user can restore their in-app purchases, reactivating any content that had previously been purchased from the same store account (Apple, Google, or Amazon). It is recommended that all apps have some way for users to trigger the restorePurchases method, even if you require all customers to create accounts. The restorePurchases method should not be triggered programmatically, since it may cause OS level sign-in prompts to appear, and should only be called from some user interaction (e.g. tapping a "Restore" button.)

syncPurchases()method 
public function syncPurchases(success:Function = null, failure:Function = null):void

syncPurchases is a method we provide in our SDK which allows you to programmatically trigger a restore. This method, much like restorePurchases, reactivates any content that had previously been purchased from the same store account (Apple, Google, or Amazon).

  • syncPurchases is typically used for migrating subscriptions
  • Since this method simulates restoring a purchase, there is a risk of transferring or aliasing an anonymous user

Parameters

success:Function (default = null) — Success callback with signature function( customerInfo:CustomerInfo ):void
 
failure:Function (default = null) — Error callback with signature function( error:PurchaseError ):void
