Packagecom.distriqt.extension.notifications
Classpublic class Notifications
InheritanceNotifications flash.events.EventDispatcher

This extension allows access to native local notifications, as opposed to push notifications. These notifications are displayed locally in the notification area of the device, generally the pull down area at the top of the device.

For detailed documentation on using this class see the "Getting Started" guide here

public function authorisationStatus():String

Returns the current authorisation status of the device. This status indicates whether the user has authorised or denied access to notifications.

The NOT_DETERMINED status is returned when your application is first installed and you have not yet requested authorisation.

Returns
String — A value from the AuthorisationStatus constants
cancel()method 
public function cancel(id:int):void

Cancel a previously shown notification. If it's transient, the view will be hidden. If it's persistent, it will be removed from the status bar. If it has been scheduled to be displayed at a delayed interval, it will be removed.

Note: iOS < 10 can only remove scheduled notifications. Once a notification is displayed in the notification center it can not be individually removed. You can remove all notifications on iOS using the cancelAll function. This is no longer applicable for iOS 10, we now remove notifications both scheduled and displayed.

Parameters

id:int — The id of the notification

See also

notify()method 
public function notify(notification:Notification):void

Post a notification to be shown in the status bar.

On Android, if a notification with the same id has already been posted by your application and has not yet been canceled, it will be replaced by the updated information. On iOS, a new notification will always be displayed.

Parameters

notification:Notification — A Notification object describing what to show the user. Must not be null. You should use the NotificationBuilder class to generate this object.


Example
Showing a simple notification 
         Notifications.service.notify(
            new NotificationBuilder()
                .setAlert( "A simple alert" )
                .setTitle( "A simple notification" )
                .setBody( "The body of the notification" )
                .build()
         );

register()method 
public function register():void

Registers to receive notifications.

This is important mainly if you expect to receive startup notifications. If the application was started via a local notification, you will receive a notification selected event immediately after calling this function so make sure you've added a listener for this event prior to calling this function.

From iOS 8, calling this function also performs the necessary registration for notifications permission, which may display a popup dialog requesting permission from the user to display notifications.

See also

com.distriqt.extension.notifications.events.NotificationEvent
requestAuthorisation()method 
public function requestAuthorisation():Boolean

This function triggers the authorisation request dialog. This dialog will only be displayed once to a user on iOS so make sure you have adequately informed your user why you are requesting authorisation.

Note calling register at any point will trigger the request authorisation dialog as well so make sure you have implemented your authorisation logic before calling register

Returns
Boolean — true if the request was started successfully and false otherwise
