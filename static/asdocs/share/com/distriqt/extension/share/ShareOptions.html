arrowDirectionproperty
public var arrowDirection:int

Controls the allowed directions for the popover dialog. This is only used in situations where the share dialog is presented as a popover dialog (currently only on an iPad).


Example
Position to the left or right of a button located at position: 
         var options:ShareOptions = new ShareOptions();
         options.position = new Rectangle( 100, 100, 200, 50 );
         options.arrowDirection = ShareOptions.ARROWDIRECTION_LEFT | ShareOptions.ARROWDIRECTION_RIGHT;
         
         Share.service.share( "Share text", null, "http://airnativeextensions.com", options );
autoScaleproperty 
public var autoScale:Boolean = false

The autoScale feature allows you to specify position and size based on the devices dip which will automatically scale the popover based on the device display resolution.

For example, the following would display a popover of width 200 pixels and height 200 pixels on an iPhone 3GS (non-retina) positioned on a box of 20x20 pixels at 150,100. However on an iPhone 5 (retina) this would appear as a box of 400 pixels positioned on a box of 40x40 at 300,200. 

         options.autoScale = true;
         options.position = new Rectangle( 150, 100, 20, 20 );
         options.size = new Rectangle( 200, 200 );

The default is false, allowing the position and size to align with the AIR pixel coordinate system.

cacheBitmapInternallyproperty 
public var cacheBitmapInternally:Boolean = true

If true then the native extension will create an internal copy of any image data to make sure the data is preserved in memory while the share process is executed.

If false you must make sure any BitmapData references passed to the extension are kept in memory for the entire share process execution.

excludedActivityTypesproperty 
public var excludedActivityTypes:Array

iOS excluded activity types when using the UIActivityViewController i.e. When calling share or shareFile with useActivityController set to true This property contains an array of strings, each of which corresponds to the value you would find in the activityType parameter of a UIActivity object. Each string you specify indicates a service that you do not want displayed to the user. You might exclude services that you feel are not suitable for the content you are providing. For example, you might not want to allow the user to print a specific image. If the value of this property is empty or null, no services are excluded. Some predefined values are available in the ActivityType class

See also

com.distriqt.extension.share.ActivityType
packageNameproperty 
public var packageName:String

On Android, this allows selection of a specific application to limit the sharing to.


Example
Allow sending to Instagram only: 
         var options:ShareOptions = new ShareOptions();
         options.packageName = "com.instagram.android";
         
         Share.service.showOpenIn( imagePath, "Instagram File", "image/, options );
positionproperty 
public var position:Rectangle

This rectangle specifies an area on the screen that the popover will be "attached" to. The native popover will best fit in order to associate the content with this area. Generally this should the be size and location of a button that was pressed by the user to initiate this popover.

showOpenInproperty 
public var showOpenIn:Boolean = false

Sharing file actions will allow an additional action to open the file in an application, effectively providing a native option to call showOpenIn with the same options as passed to shareFile

iOS only

useChooserproperty 
public var useChooser:Boolean = true

On Android you can present an application 'chooser' to the user to allow them to select the application or you can just use the default application.

If this value is true then the extension will display a 'chooser' to the user.

Note: If the user has no default application set then they may get a 'chooser' displayed even if this setting is false. Also if there is only one application that can handle this type the chooser may not be displayed even if this setting is true

UTIproperty 
public var UTI:String

On iOS this is similar to a mimeType, this is the 'type' of the file that is being shared using the showOpenIn. It indicates which apps are capable of opening the file.


Example
Allow sending to Instagram: 
         var options:ShareOptions = new ShareOptions();
         options.UTI = "com.instagram.photo";
         
         Share.service.showOpenIn( imagePath, "Instagram File", "image/, options );
