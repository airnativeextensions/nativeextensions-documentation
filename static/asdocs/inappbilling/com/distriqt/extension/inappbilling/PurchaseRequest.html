Property Detail
applicationUsernameproperty
public var applicationUsername:String

If your users have an account with you, in addition to their App Store accounts, provide this additional piece of information when requesting payment.

We suggest you hash this value before passing any clear text user identifier value.

iOS notes

By way of illustration, consider the following two examples. In the normal case, many different users on your server each buy coins to use in your game, and each user pays for the purchase from a different App Store account. In contrast, it would be very unusual for a single user on your server to buy coins multiple times, paying for each purchase from a different App Store account. The App Store can't detect this kind of irregular activity on its own—it needs information from your app about which account on your server is associated with the transaction.

Android notes

If you pass this value, Google Play can use it to detect irregular activity, such as many devices making purchases on the same account in a short period of time. Do not use the developer ID or the user's Google ID for this field. In addition, this field should not contain the user's ID in cleartext. We recommend that you use a one-way hash to generate a string from the user's ID and store the hashed string in this field.

developerPayloadproperty 
public var developerPayload:String
Deprecated: Consider using applicationUsername instead

This is a developer payload string that you can use to send supplemental information about an order (it can be an empty string).

Typically, this is used to pass in a string token that uniquely identifies this purchase request.

Unfortunately Google have deprecated this functionality and it no longer functions in the latest version of their library. You now can set a developer payload when consuming or finishing a purchase instead. See this link for more information.
