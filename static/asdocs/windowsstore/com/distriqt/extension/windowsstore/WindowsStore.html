getProducts()method 
public function getProducts(productIds:Array = null):Boolean

Retrieves a list of products from the store.

These must be setup in the store previously and you must have at least one product defined.

You must call and wait for a response from this function before attempting to make a purchase, even if you aren't going to use the product information returned.

Events
  • ProductEvent.GET_PRODUCTS_COMPLETE: Dispatched when the retrieval completed successfully
  • ProductEvent.GET_PRODUCTS_CANCELLED: Dispatched when the retrieval was cancelled
  • ProductEvent.GET_PRODUCTS_ERROR: Dispatched when an error occurred

Parameters

productIds:Array (default = null) — An array of store product ids. If empty (or null) the extension will attempt to retrieve any available active products from the store.

Returns
Boolean — true if the process started successfully and false if there was an error
getPurchases()method 
public function getPurchases():Boolean

Queries the server for a list of purchases that are stored by the server.

Once complete this will dispatch either ProductEvent.GET_PRODUCTS_COMPLETE or ProductEvent.GET_PRODUCTS_ERROR depending on the success of the call to the store.

Returns
Boolean — true if the query started successfully and false if there was an error initiating the query.

public function getStoreId(accessToken:String, userId:String, type:String = collection):Boolean

Retrieve the Microsoft Store ID key for use with the Microsoft Store collection and purchase API

See the documentation on tokens: https://docs.microsoft.com/en-au/windows/uwp/monetize/view-and-grant-products-from-a-service#step-3

Parameters

userId:String — A parameter string that identifies the current user in the context of your services
 
type:String (default = collection) — Either 'collection` for the collections API key or 'purchase' for the purchase API key

Returns
Boolean — true if successfully started and false if there was an issue or setup has not been called
