developerPayloadproperty 
public var developerPayload:String

This is a 'developer payload' string that you can use to send supplemental information about an order (it can be an empty string).

Typically, this is used to pass in a string token that uniquely identifies this purchase request.

Note: Most services have deprecated this property and is no longer in use

originalMessageproperty 
public var originalMessage:String

This will contain the actual contents of the message received by the in app billing extension. Details:

  • under Google Play it will contain the JSON formatted purchase message received from Google Play;
  • other platforms will contain the raw JSON data received from the implementation, it may contain additional information depending on the variant;
You may wish to use the additional information for specific platform features.

transactionReceiptproperty 
public var transactionReceipt:String

The receipt of the transaction.

  • Google Play this is the "order ID"
  • iOS InAppPurchases
    • Previous to iOS 7 this is populated with each transaction's individual receipt
    • From iOS 7 onwards, this will be populated with the current value of the application receipt
Note

If this value is empty on iOS > 7 you may need to refresh your application receipt (see the ApplicationReceipt class)

See also

com.distriqt.extension.inappbilling.ApplicationReceipt

STATE_DEFERREDConstant 
public static const STATE_DEFERRED:String = transaction:deferred

The purchase has been deferred. This state can occur in several situations, all of which mean the purchase has been delayed and will be completed at a later point. It may fail or succeed so you should not deliver your product at this stage. This state includes situations such as:

  • awaiting parental approval;
  • user has selected a manual payment (cash) at a physical store;
