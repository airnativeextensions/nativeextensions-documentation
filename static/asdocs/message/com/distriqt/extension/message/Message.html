This class represents the message extension.

Email Functionality

The extension allows you to send an email using the devices native user interface.

Example sending a HTML email: 

     Message.service.addEventListener( MessageEvent.MESSAGE_MAIL_ATTACHMENT_ERROR, message_errorHandler, false, 0, true );
     Message.service.addEventListener( MessageEvent.MESSAGE_MAIL_COMPOSE, message_composeHandler, false, 0, true );
     if (Message.isMailSupported)
     {
         var address:String = "ma@distriqt.com";
         var email:String = "samurai";
         var htmlEmail:String = "<!DOCTYPE html><html><body><p>A test html email with a <a href='http://distriqt.com'>link</a></p></body></html>";
     
         Message.service.sendMailWithOptions( "test", htmlEmail, address, "", "", [], true ); 
     }
     
     private function message_errorHandler( event:MessageEvent ):void 
     {
         trace( event.type +"::"+ event.details );
     }
     private function message_composeHandler( event:MessageEvent ):void 
     {
         trace( event.type +"::"+ event.details );
     }
SMS Functionality

On Android you must add the following permissions to your manifest file 

	 <uses-permission android:name="android.permission.WRITE_SMS" /> 
	 <uses-permission android:name="android.permission.SEND_SMS" /> 
	 <uses-permission android:name="android.permission.READ_SMS" />
Receiving an SMS

On Android you can get information about an SMS when the device receives one. On iOS it is not possible to receive this information as this information is only passed to the default messaging application.

You'll need to add the following additional permission to receive SMS to your manifest additions. (Note the application node contents needs to be added to your existing application node, if you already have one. Having 2 application nodes will break the application). 

	 <uses-permission android:name="android.permission.RECEIVE_SMS" />
	 
	 <application>
	 	<receiver android:name="com.distriqt.extension.message.receivers.MessageSMSReceiver" android:exported="true" > 
	 		<intent-filter android:priority="1000"> 
	 			<action android:name="android.provider.Telephony.SMS_RECEIVED" />
	 		</intent-filter> 
	 	</receiver>
	 </application>
	 
     Message.service.addEventListener( MessageSMSEvent.MESSAGE_SMS_RECEIVED, smsReceivedHandler );
     
     private function smsReceivedHandler( event:MessageSMSEvent ):void
     {
         trace( "SMS has been received" );
         trace( event.sms.address + ": " + event.sms.message );
     }
Sending an SMS

Sending an SMS can be done in one of 2 ways, with or without a UI, though only on Android. On iOS you are limited to using the native UI through the messaging application.

Firstly construct an SMS to send: 

     var sms:SMS = new SMS();
     sms.address = "0444444444"; // The destination phone number
     sms.message = "SMS Message from actionscript"; // The message to send

Then send this object using one of the send SMS functions. If you use the UI function the UI will be populated with the above information 

     Message.service.addEventListener( MessageSMSEvent.MESSAGE_SMS_SENT, smsSentHandler );
     Message.service.addEventListener( MessageSMSEvent.MESSAGE_SMS_CANCELLED, smsCancelledHandler );
     Message.service.sendSMSWithUI( sms );
     
     private function smsSentHandler( event:MessageSMSEvent ):void
     {
         trace( "The SMS has been sent" );
     }
     private function smsCancelledHandler( event:MessageSMSEvent ):void
     {
      trace( "The user cancelled the SMS" );
     }

isMailSupportedproperty 
isMailSupported:Boolean  [read-only]

This checks that the device you are using has the ability to send an email. Generally when this returns false it is because the device doesn't have a mail client or an email account setup on the device.

Use this to check if you should expect sendMail to complete successfully.

iOS: Double check in Settings / "Mail, Contacts ..." that there's a valid email account.

Please note that you should incorporate this check into your application and inform users if they don't have the ability to send email from their device.

true if email messaging is supported

sendMail()method 
public function sendMail(subject:String, body:String, toRecipients:String):void

Sends an email using the native "compose mail" user interface. The parameters of the UI are prepopulated with the information passed into this function.

Recipients are specified as a comma separated string of email addresses, and can be an empty string if not required.

Supported OS: default, iOS, Android

Parameters

subject:String — The subject of the email to prepopulate
 
body:String — The body content of the email
 
toRecipients:String — The list of to addresses

Sends an email using the native "compose mail" user interface. The parameters of the UI are prepopulated with the information passed into this function.

Recipients are specified as a comma separated string of email addresses, and can be an empty string if not required.

Note: Attachments are not yet supported in the default implementation.

HTML Content

The way the HTML content is rendered and the tags supported is greatly dependant on the email application used to send the email. The following list are the generallly accepted tags, there are a few more on iOS (including table support) but you are better off sticking to only using the following. If you require additional tags, we suggest you save your output as html and attach it as a file.

  • <a href="...">
  • <b>
  • <big>
  • <blockquote>
  • <br>
  • <cite>
  • <dfn>
  • <div align="...">
  • <em>
  • <font size="..." color="..." face="...">
  • <h1>
  • <h2>
  • <h3>
  • <h4>
  • <h5>
  • <h6>
  • <i>
  • <img src="...">
  • <p>
  • <small>
  • <strike>
  • <strong>
  • <sub>
  • <sup>
  • <tt>
  • <u>

Supported OS: default, iOS, Android

Parameters

subject:String — The subject of the email to prepopulate
 
body:String — The body content of the email
 
toRecipients:String — The list of to addresses
 
ccRecipients:String — The list of cc addresses (carbon copy)
 
bccRecipients:String — The list of bcc addresses (blind carbon copy)
 
attachments:Array (default = null) — The attachments array. This should be an array of MessageAttachment objects. Default: null i.e. no attachments
 
isHTML:Boolean (default = true) — If true the body string is rendered as HTML in the email content, if false it's treated as plain text.
 
useChooser:Boolean (default = true) — If true on Android a dialog will be displayed to allow the user to choose the mail application to use.

share()method 
public function share(text:String, image:BitmapData = null, url:String, options:ShareOptions = null):void

Displays a share action that allows the user to choose another application to share the specified data with.

You should always check isShareSupported before calling this function.

On iPads the dialog displayed allowing the user to select the destination application should be shown in a popover. The position and size of this popover is controlled with the options parameter. If null it is displayed modally.

Parameters

text:String — The text content to share
 
image:BitmapData (default = null) — Image data to share, if null no image is shared
 
url:String — A url to share, if empty ("") no url is shared
 
options:ShareOptions (default = null) — This specifies the position of the dialog when presented on iPads.

message:mail:compose Event  
Event Object Type: com.distriqt.extension.message.events.MessageEvent
MessageEvent.type property = com.distriqt.extension.message.events.MessageEvent.MESSAGE_MAIL_COMPOSE

Returns information about the send email dialog created on iOS, or just information about the dialog returning.

This is not possible on Android as the email is handled by many possible external applications which all return different result/error codes (or none) and depend on user choices. Android will return complete but that is the only event.

Note: Under android to receive the complete event you must add in an activity to your application manifest additions as below: 

		 <application>
		   <activity android:name="com.distriqt.extension.message.activities.SendMailActivity" android:theme="@android:style/Theme.Translucent.NoTitleBar" ></activity>
		 </application>

If you don't add this the extension will still function you just won't get the complete notification

The #details of this event this contains one of the following MESSAGE_MAIL_COMPOSE_types.
