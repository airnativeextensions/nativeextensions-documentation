public function sendSMS(sms:SMS, subscriptionId:int = -1):void

Sends an sms directly without any user interface. This is only available on Android devices. iOS must use the UI based functionality.

Android

Requires the SEND_SMS permission to be added to your application manifest additions: 

		 <uses-permission android:name="android.permission.SEND_SMS" />

Parameters

sms:SMS — The SMS message to send
 
subscriptionId:int (default = -1) — The identifier of a subscription to use to send the SMS, uses the default if not specified

sendSMSWithUI
sendSMSWithUI()method 
public function sendSMSWithUI(sms:SMS, useChooser:Boolean = false):void

Send an SMS by opening up the user interface and allowing the user to edit the message before sending.

Firstly construct an SMS to send: 

         var sms:SMS = new SMS();
         sms.address = "0444444444"; // The destination phone number
         sms.message = "SMS Message from actionscript"; // The message to send

Then pass this SMS to the parameter of this function. The UI will be populated with the above information 

         Message.service.addEventListener( MessageSMSEvent.MESSAGE_SMS_SENT, smsSentHandler );
         Message.service.addEventListener( MessageSMSEvent.MESSAGE_SMS_CANCELLED, smsCancelledHandler );
         Message.service.sendSMSWithUI( sms );
                  private function smsSentHandler( event:MessageSMSEvent ):void
         {
             trace( "The SMS has been sent" );
         }
         private function smsCancelledHandler( event:MessageSMSEvent ):void
         {
          trace( "The user cancelled the SMS" );
         }

Parameters

sms:SMS — The SMS message to send
 
useChooser:Boolean (default = false) — Android only: If true the user will be presented a choice on the application to use otherwise the default SMS application will be used
