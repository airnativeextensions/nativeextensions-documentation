EULERConstant
public static const EULER:String = euler

The simplest and most problematic return values. Only use these if you really cannot use one of the other formats.

We attempt to correct these values by using quaternions internally and attempting to overcome the Gimbal lock issue.

You will encounter odd motion in the form of a Gimbal lock using these values where you'll lose a degree of freedom. http://en.wikipedia.org/wiki/Gimbal_lock

EULER_RAWConstant 
public static const EULER_RAW:String = euler:raw

This returns the raw Euler values from the native process. Generally these are quite unusable but are provided as an option if required.

You will encounter odd motion in the form of a Gimbal lock using these values where you'll lose a degree of freedom. http://en.wikipedia.org/wiki/Gimbal_lock This will happen when the phone is held vertically.

QUATERNIONConstant 
public static const QUATERNION:String = quaternion

This is the best format to retrieve the device rotation information.

Quaternions are a number system extending complex numbers, and are sort of similar to a vector in concept in three dimensions. They provide a very convenient way to represent orientations and rotations in three dimensions.

Read more on quaternions here:

  • http://en.wikipedia.org/wiki/Quaternions_and_spatial_rotation
  • http://content.gpwiki.org/index.php/OpenGL%3aTutorials%3aUsing_Quaternions_to_represent_rotation
