IMMERSIVEConstant 
public static const IMMERSIVE:String = immersive

The Immersive mode is a true fullscreen interactive mode. System status bars and controls are hidden allowing your application to still receive touch interactions while being in the full-screen mode.

The user can reveal the system bars with an inward swipe along the region where the system bars normally appear. They will hide again after a small interval.

UI_NAVIGATION_HIDEConstant 
public static const UI_NAVIGATION_HIDE:String = ui:navigation:hide

View has requested that the system navigation be temporarily hidden.

This is an even less obtrusive state than that called for by SYSTEM_UI_FLAG_LOW_PROFILE; on devices that draw essential navigation controls (Home, Back, and the like) on screen, UI_NAVIGATION_HIDE will cause those to disappear. This is useful (in conjunction with the FULLSCREEN window flags) for displaying content using every last pixel on the display.

There is a limitation: because navigation controls are so important, the least user interaction will cause them to reappear immediately. When this happens, both this flag and SYSTEM_UI_FLAG_FULLSCREEN will be cleared automatically, so that both elements reappear at the same time.

UI_NAVIGATION_LOW_PROFILEConstant 
public static const UI_NAVIGATION_LOW_PROFILE:String = ui:navigation:lowprofile

View has requested the system UI to enter an unobtrusive "low profile" mode.

This is for use in games, book readers, video players, or any other "immersive" application where the usual system chrome is deemed too distracting. In low profile mode, the status bar and/or navigation icons may dim.
