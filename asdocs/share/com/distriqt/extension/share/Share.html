share()method 
public function share(text:String, image:BitmapData = null, url:String, options:ShareOptions = null):void

Displays a share action that allows the user to choose another application to share the specified data with.

You should always check isShareSupported before calling this function.

iOS

On iOS this uses the UIActivityViewController to display a list of activities that support the supplied data. It also provides actions such as:

  • Save image
  • Copy
  • Print
  • Add to Reading List

On iPads the dialog displayed allowing the user to select the destination application should be shown in a popover. The position and size of this popover is controlled with the options parameter. If null it is displayed modally.

Android

Unfortunately different applications on Android handle the url and text different when they are both supplied to the intent.

We suggest that if you are wanting a url and text to appear in the content of a message that you simply combine the two into the text parameter: 

         Share.service.share( "something to share " + "http://airnativeextensions.com" );

This is the most consistent method if you are not concerned about the url being separated.

Parameters

text:String — The text content to share
 
image:BitmapData (default = null) — Image data to share, if null no image is shared
 
url:String — A url to share, if empty ("") no url is shared
 
options:ShareOptions (default = null) — Additional sharing options including the position of the dialog when presented on iPads.

shareFile()method 
public function shareFile(path:String, name:String, mimeType:String, options:ShareOptions = null, useActivityViewController:Boolean = false):void

Displays a share action dialog to that allows the user to choose another application services that accept the specified file.

You should always check isShareSupported before calling this function.

iOS

On iOS this uses the UIActivityViewController to display a list of services that support the supplied data.

On iPads the dialog displayed allowing the user to select the destination application should be shown in a popover. The position and size of this popover is controlled with the options parameter. If null it is displayed modally.

If options.showOpenIn is true then the dialog will display an "Open in ..." action which will trigger display of the showOpenIn dialog.

Parameters

path:String — The path of a file to share
 
name:String — The name of the file to use when sharing
 
mimeType:String — The mime type of the file (is used to limit the displayed applications)
 
options:ShareOptions (default = null) — Additional sharing options including the position of the dialog when presented on iPads
 
useActivityViewController:Boolean (default = false) — This controls whether this function on iOS uses the activity view controller or the document interaction controller

showOpenIn()method 
public function showOpenIn(path:String, name:String, mimeType:String, options:ShareOptions = null):void

Displays an open in dialog that allows the user to choose another application to open the specified file with.

You should always check isShareSupported before calling this function.

iOS

On iOS this uses the UIDocumentInteractionController to display a list of applications that can open the specified file.

On iPads the dialog displayed allowing the user to select the destination application should be shown in a popover. The position and size of this popover is controlled with the options parameter. If null it is displayed modally.

Parameters

path:String — The path of a file to share
 
name:String — The name of the file to use when sharing
 
mimeType:String — The mime type of the file (is used to limit the displayed applications)
 
options:ShareOptions (default = null) — Additional sharing options including the position of the dialog when presented on iPads

share:complete Event  
Event Object Type: com.distriqt.extension.share.events.ShareEvent
ShareEvent.type property = com.distriqt.extension.share.events.ShareEvent.COMPLETE

Dispatched when the share / open is complete.

On iOS you'll be able to determine the activity used to share the content via the activityType variable on this event.

On Android this event is dispatched when the activity finishes, either as a success or a failure. Android doesn't return information about the success of the activity.
