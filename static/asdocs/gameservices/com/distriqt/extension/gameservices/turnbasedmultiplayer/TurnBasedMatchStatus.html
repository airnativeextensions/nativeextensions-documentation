MATCH_STATUS_ACTIVEConstant 
public static const MATCH_STATUS_ACTIVE:int = 1

The match has started and is active. Indicates that a new match is created and your game can let match participants take their turns.

MATCH_STATUS_AUTO_MATCHINGConstant 
public static const MATCH_STATUS_AUTO_MATCHING:int = 0

This state indicates that there are still empty auto-match slots available and your game is waiting for an auto-matched player to join. Your game cannot let other match participants take turns while the match is in this state.

MATCH_STATUS_CANCELEDConstant 
public static const MATCH_STATUS_CANCELED:int = 4

Indicates that a match ended before its normal completion. This might occur, for example, if a user who was invited to the match declined the invitation or if a participant explicitly cancels the match. Game services allows participants to cancel the match at any point after joining a match (if you game interface supports this action).

MATCH_STATUS_COMPLETEConstant 
public static const MATCH_STATUS_COMPLETE:int = 2

Indicates that a match has been played to completion (for example, a player has won the match). Once a player signals that the game is finished, games services sends a notification to all match participants to inform them that the match is over. Players have a chance to save their final game data but may not modify the game state further.

MATCH_STATUS_EXPIREDConstant 
public static const MATCH_STATUS_EXPIRED:int = 3

Indicates that a match has expired. A match expires when a user does not respond to an invitation or turn notification for two weeks. A match also expires in one day if there is an empty auto-match slot available but games services cannot find a user to auto-match.
