CUTOUT_ALWAYSConstant
public static const CUTOUT_ALWAYS:String = cutout_always

Content is always allowed to extend into the cutout areas. If your app targets SDK 35 and is running on an Android 15 device, this is the only allowed mode for non-floating windows to ensure an edge-to-edge display.

CUTOUT_DEFAULTConstant 
public static const CUTOUT_DEFAULT:String = cutout_default

The window is allowed to extend into the cutout area, only if the cutout is fully contained within a system bar. Otherwise, the window is laid out such that it does not overlap with the cutout area. In practice, this means that if the window did not set fullscreen it can extend into the cutout area in portrait if the cutout is at the top edge. Similarly for {

CUTOUT_NEVERConstant 
public static const CUTOUT_NEVER:String = cutout_never

The window is never allowed to overlap with the cutout area. If your app targets SDK 35, this is interpreted as CUTOUT_ALWAYS

CUTOUT_SHORT_EDGESConstant 
public static const CUTOUT_SHORT_EDGES:String = cutout_short_edges

The window is always allowed to extend into the cutout areas on the short edges of the screen. The window will never extend into a cutout area on the long edges of the screen. If your app targets SDK 35, this is interpreted as CUTOUT_ALWAYS
